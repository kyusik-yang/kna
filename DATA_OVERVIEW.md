## 2. 법안의 여정 (Life of a Bill)

각 법안은 아래 경로를 따라 진행되며, 각 단계의 날짜가 기록됩니다.

```
  발의(ppsl_dt)
    │
    ▼
  소관위 회부(committee_dt)  ··· 평균 3.6일 (중위 1일)
    │
    ▼
  소관위 상정(cmt_present_dt)
    │
    ▼
  소관위 심사/의결(cmt_proc_dt)
    │   ├─ 대안반영폐기 ─── 다른 법안에 내용 반영 후 폐기
    │   ├─ 원안가결 ─────── 원안 그대로 통과
    │   ├─ 수정가결 ─────── 수정 후 통과
    │   └─ 폐기/부결 ────── 여기서 종료
    ▼
  법사위 회부(law_submit_dt)  ◄── Phase 2에서 채워질 필드
    │
    ▼
  법사위 심사(law_proc_dt)
    │
    ▼
  본회의 상정/표결(rgs_rsln_dt)
    │   ├─ 원안가결
    │   ├─ 수정가결
    │   └─ 부결
    ▼
  정부이송(gvrn_trsf_dt)
    │
    ▼
  공포(prom_dt)  ──── 법률로 확정
```

```
              └──────┘    └──────────┘     └──────────┘
              ┌──────┐    ┌──────────┐     ┌──────────┐

              └──────┘    └──────────┘     └──────────┘
              ┌──────┐    ┌──────────┐     ┌──────────┐
  위원장      │   635│    │ 99.7%    │     │ 99.7%    │  ◄ 위원장안은 사실상 전수 통과
              └──────┘    └──────────┘     └──────────┘

  * "통과"는 대안반영폐기 포함 (법안 내용이 다른 법안에 반영되어 실질적으로 통과)
  * "가결"은 원안가결 + 수정가결만 (해당 법안 자체가 법률로 확정)
```

## 12. Phase 2 완료 후 추가되는 정보

Phase 2 (BILLINFODETAIL)가 완료되면 법안별 **전 생애주기 타임스탬프**가 채워집니다:

| 추가 필드 | 설명 | 분석 활용 |
|-----------|------|----------|
| `jrcmit_cmmt_dt` | 소관위 회부일 (정밀) | 단계별 소요일 계산 |
| `law_cmmt_dt` | 법사위 회부일 | 법사위 병목 분석 |
| `law_prsnt_dt` | 법사위 상정일 | 법사위 내 지연 측정 |
| `law_proc_rslt` | 법사위 처리결과 | 법사위 gatekeeping 분석 |
| `rgs_prsnt_dt` | 본회의 상정일 | 본회의 진입 시점 |
| `rgs_rsln_dt` | 본회의 의결일 | 최종 의결 타이밍 |
| `rgs_conf_rslt` | 본회의 결과 | 정밀 결과 분류 |
| `gvrn_trsf_dt` | 정부이송일 | 후속 절차 소요 |
| `prom_dt` | 공포일 | 법률 확정 시점 |
| `prom_no` | 공포번호 | 법률 식별 |

위성 테이블:
- `committee_meetings`: 법안별 위원회 회의 내역 (상정, 소위 심사, 의결 등)
- `judiciary_meetings`: 법안별 법사위 회의 내역
